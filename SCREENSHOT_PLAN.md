# Screenshot plan

No final store screenshot is included or claimed by this repository. Capture only after the release-QA matrix passes and the owner approves the build and copy.

## Draft capture set

- PWA/browser QA: 390×844 and 360×740 portrait.
- Store portrait masters: 1290×2796, 1242×2688, and 1080×1920 pixels. Treat these as working capture targets; the owner must verify current required sizes, count, aspect ratio, and device classes in each chosen portal before submission.
- If a store requires tablet, landscape, desktop, wearable, TV, or feature-graphic assets, mark them **PLACEHOLDER — target not chosen** until the owner selects that platform.

## Required scenes

1. Workshop with pantry, charged Gather control, and readable core resources.
2. Active cauldron with a brew in progress.
3. Ready brew with Collect available.
4. Village orders with a deliverable request.
5. Workshop upgrades with at least one affordable choice.
6. Journal achievements/progress.
7. Optional Moonlight Market only if the frame visibly says `PROTOTYPE` / `simulated` and listing copy also discloses that no real ad or billing system is connected.

## Safe capture procedure

Use a fresh synthetic local save with no personal data. Run the exact-size loop and confirm no overflow, console errors/warnings, clipped safe areas, focus artifacts, debug UI, browser chrome, notifications, account names, file paths, or other apps appear. Keep sound muted. Capture real rendered UI; do not fabricate ratings, reviews, prices, rewards, devices, or features. Record commit/branch, package version, viewport/device scale, browser/OS, scene state, and capture date. Crop only to a portal's verified rules and never stretch. Owner/design must approve every selected image before upload.
